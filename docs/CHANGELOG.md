# 更新日志 / Changelog

本文件记录每个发行版的变更。**每次发布前必须在顶部新增一个版本段落，写清楚本次发行「新增 / 删除 / 调整」了什么功能**（无对应项可写「无」）。发行说明（GitHub Release notes）由 CI 从本文件对应版本段落自动生成。

格式：`## v<版本> — <YYYY-MM-DD>`，下设 `### 新增`、`### 调整`、`### 删除`（按需保留）。

## v1.2.6 — 2026-06-29

### 新增
- **🌟 局域网剪贴板同步（开发人员模式）—— 本次核心新功能**：新增「剪贴板同步」页（开发工具组，开发人员模式可见）。
  - **设备发现**：通过 UDP 多播信标自动发现同一局域网内**同样运行 OwO! WinDeploy** 的在线设备（仅暴露在线状态，不经此通道传任何剪贴板数据）；逐网卡多播 + 定向子网广播，可在卡片上**选择监听网卡**（应对多虚拟网卡机器），自动发现不通时可**手动按 IP 连接**兜底。
  - **PIN 配对**：发起方点「邀请」生成并显示 6 位 PIN，对方输入；经 PBKDF2 派生密钥 + 双向 HMAC 挑战应答校验（**PIN 永不上线**），通过后派生 **AES-256-GCM** 会话密钥，建立端到端加密链路。
  - **共享文本 + 图片**：共享列表双向同步、预览、手动新增文本、**删除即跨设备同步**、一键复制到本机；图片带缩略图 + 自适应预览，**点击可在独立窗口查看大图**。
  - **可选行为（默认关闭，更安全）**：「自动镜像到本机剪贴板」（真·同步）与「历史记录落盘保存」（重启保留）两个开关由用户自选；否则剪贴板内容仅存内存、退出即清。
  - **设备上限**：开源版每个共享最多 **2 台设备**；传输层已抽象为可替换实现，付费版可经服务器中转解除上限并支持跨局域网。
  - 活动日志全面支持中 / 英 / 德三语。
- **便携数据目录**：所有本机运行时数据（日志 / 设置 / 图标缓存 / FTP / Cloudflare / 剪贴板配置 / 报告 / 崩溃日志）统一写到 exe 旁的 `data/`（`AppPaths.DataRoot`），整个应用成为可整体移动 / 删除的自包含文件夹（`data/` 已加入 `.gitignore`）；旧 `%LOCALAPPDATA%\WinDeploy` 数据首次启动时**询问后迁移**。
- **同步本机已安装软件为方案**：配置同步页可扫描本机已装的清单软件并存为可复用 profile，换机在「软件安装中心」一键复刻；套用配置 / 生成 SSH 密钥前新增确认弹窗。

### 调整
- **SSH known_hosts 改为合并**：套用 ssh 配置时 `known_hosts` 由覆盖改为**合并**，只追加本地缺失的条目，不再覆盖你已信任的主机密钥。

### 修复
- **FTP 传输不再卡界面**：FTP 文件传输移至后台线程，避免大文件传输时 GUI 冻结。

### 删除
- 无。

## v1.2.5 — 2026-06-26

### 新增
- **单实例限制**：应用仅允许启动一个实例；再次双击不再多开进程，而是把已运行的窗口（含最小化 / 托盘）唤回前台。
- **存为方案**：软件安装中心新增「存为方案」按钮，把当前勾选的软件组合保存为可复用的 profile（写入 `catalog/profiles/<名称>.json`，已存在可确认覆盖），之后从「方案」下拉一键套用。
- **智能体会话记忆采集**：「采集本机配置」时，Claude（`projects\*\memory`、`memory`、`commands`、`agents`）与 Codex（`memory`）的会话记忆随其配置一并导出；大体积的 `*.jsonl` 会话记录与缓存被排除。
- **环境配置恢复 · 漂移预览**：「环境配置恢复」前先干跑比对，逐文件标记 新增 / 变更 / 相同 并列出将创建或覆盖的文件，避免盲目覆盖；无可恢复内容或已一致时直接提示。
- **部署报告**：批量安装实际执行后生成 HTML 报告（逐项 状态 / 耗时 / 信息 + 成功 / 失败 / 跳过 汇总），写入应用数据目录的 `reports/` 并询问是否打开。
- **安装前系统还原点**：设置页「安装安全」新增开关，批量安装前创建 Windows 系统还原点（`Checkpoint-Computer`）；创建失败（需管理员且启用系统还原）时询问是否仍继续。
- **定时采集**：采集页新增「定时采集」开关 + 频率（每日 / 每周 / 登录时），通过 Windows 计划任务定期把本机配置无界面采集进仓库（仅非敏感）；新增无界面入口 `WinDeploy.exe --capture <仓库路径>`。
- **远程部署**：配置同步页新增「远程部署」对话框——可选择任意文件 / 文件夹作为来源，并选择传输方式 **SSH/SCP**（密钥认证，可在对端运行部署命令）或 **FTP / FTPS**（复用内置 FTP 客户端）。
- **CLI `hash` 命令**：下载便携项安装包计算 SHA256，`--write` 就地回写 `catalog.json`（保留注释 / 格式），用于补全完整性校验。
- **自动化测试基线**：新增 `WinDeploy.Core.Tests`（xUnit / .NET 10），覆盖方案解析、catalog 加载、脱敏、三语 i18n 键对齐、catalog 校验等 47 个用例。

### 调整
- 文档：第三档 TODO 标记完成；`docs/TODO.md` 同步更新实现说明。

### 删除
- 无。

## v1.2.3 — 2026-06-25

### 新增
- **硬件温度监控**：后台定时监控 CPU、GPU 及**所有已连接硬盘**（NVMe 含 USB 外置盒，免管理员；SATA / USB-SATA 需管理员）的温度，超过用户设定阈值时弹系统通知，并按界面语言（中 / 英 / 德）用系统 TTS 语音播报，**念出具体设备名与当前温度**；温度持续过高会按设定频率反复提醒。阈值与提醒频率在「设置 → 硬件温度监控」**集中统一管理**；回到应用时可在弹窗中将某设备本次运行设为忽略。
- **下载代理**：「设置 → 下载代理」可让应用内的 HTTP 下载（含 GitHub 发布、软件图标）通过代理（http / https / socks5 / socks4，支持 `user:pass@` 认证）。地址输入经**严格正则校验**，保存前先做**连通性测试，仅在测试通过后才保存生效**；可勾选启用 / 取消勾选关闭。
- **重置所有设置与数据**：「设置」页保存按钮右侧新增红色重置按钮，需**三重确认**（两次风险警告弹窗 + 输入短语 `I understand the risk`，大小写不敏感）；确认后清除全部设置与应用数据目录并自动重启为初始状态。
- **托盘常驻图标提示**：首次开启「始终在系统托盘显示常驻图标」时弹出一次性确认提示，并引导在 Windows 11 的「^」溢出区找到该图标。

### 调整
- **SMART 磁盘详情属性名完成三语国际化**：物理磁盘 SMART 弹窗中的属性名（读取错误率 / 通电时间 / 温度 等）改为跟随界面语言显示（此前为硬编码中文）。
- 文档：`docs/DESIGN.md` 的仓库结构示例由旧名 `win-provision/` 更正为 `owo-win-deployer/`。

### 删除
- 无。
