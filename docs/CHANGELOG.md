# 更新日志 / Changelog

本文件记录每个发行版的变更。**每次发布前必须在顶部新增一个版本段落，写清楚本次发行「新增 / 删除 / 调整」了什么功能**（无对应项可写「无」）。发行说明（GitHub Release notes）由 CI 从本文件对应版本段落自动生成。

格式：`## v<版本> — <YYYY-MM-DD>`，下设 `### 新增`、`### 调整`、`### 删除`（按需保留）。

---

## v1.2.4 — 2026-06-25

### 新增
- **环境 / 智能体配置采集与一键恢复**：「采集本机配置」可导出 SSH、GnuPG、git 凭据、Codex、Claude、OpenSSH 等环境配置；「配置同步」页新增「环境配置恢复」卡片，把仓库中采集的环境配置写回新机器（覆盖前自动备份，恢复的私钥会自动收紧文件权限）。
- **敏感数据导出开关**：点击「采集本机配置」时弹窗询问是否包含敏感数据；选「否」沿用脱敏规则，选「是」才导出私钥 / 令牌等原文。采集输出全部写入 `configs/<id>/`，并由 `.gitignore` 整目录忽略（个人专属，绝不入公共库）。

### 调整
- **代码审计修复与健壮性打磨**：服务运行结果判定、事件日志崩溃识别范围收窄、安装依赖失败时跳过其后继项、安装中心套用 Profile 失败提示、环境变量写入后广播 `WM_SETTINGCHANGE`、HTTP 代理应用加锁、portable 无 sha256 由警告降为信息等多处修正。
- **文档归类**：除 `README` / `README_CH` / `CLAUDE.md` 外，其余文档统一移入 `docs/`（`CHANGELOG.md`、`TODO.md` 与既有 `DESIGN.md`）；发布流水线改为从 `docs/CHANGELOG.md` 生成 Release notes。

### 删除
- 移除未使用的占位页 `PlaceholderView` 及其 ViewModel / DataTemplate。

## v1.2.3 — 2026-06-25

### 新增
- **硬件温度监控**：后台定时监控 CPU、GPU 及**所有已连接硬盘**（NVMe 含 USB 外置盒，免管理员；SATA / USB-SATA 需管理员）的温度，超过用户设定阈值时弹系统通知，并按界面语言（中 / 英 / 德）用系统 TTS 语音播报，**念出具体设备名与当前温度**；温度持续过高会按设定频率反复提醒。阈值与提醒频率在「设置 → 硬件温度监控」**集中统一管理**；回到应用时可在弹窗中将某设备本次运行设为忽略。
- **下载代理**：「设置 → 下载代理」可让应用内的 HTTP 下载（含 GitHub 发布、软件图标）通过代理（http / https / socks5 / socks4，支持 `user:pass@` 认证）。地址输入经**严格正则校验**，保存前先做**连通性测试，仅在测试通过后才保存生效**；可勾选启用 / 取消勾选关闭。
- **重置所有设置与数据**：「设置」页保存按钮右侧新增红色重置按钮，需**三重确认**（两次风险警告弹窗 + 输入短语 `I understand the risk`，大小写不敏感）；确认后清除全部设置与应用数据目录并自动重启为初始状态。
- **托盘常驻图标提示**：首次开启「始终在系统托盘显示常驻图标」时弹出一次性确认提示，并引导在 Windows 11 的「^」溢出区找到该图标。

### 调整
- **SMART 磁盘详情属性名完成三语国际化**：物理磁盘 SMART 弹窗中的属性名（读取错误率 / 通电时间 / 温度 等）改为跟随界面语言显示（此前为硬编码中文）。
- 文档：`docs/DESIGN.md` 的仓库结构示例由旧名 `win-provision/` 更正为 `owo-win-deployer/`。

### 删除
- 无。

## v1.2.2 — 2026-06-25

### 新增
- **能效管理页面**（「系统」分组）：通过 `powercfg` 一键切换 节能 / 平衡 / 高性能 / 卓越性能 电源调度策略；列出本机全部电源计划并可逐项应用；隐藏的「卓越性能」可一键解锁并启用。全程无需管理员权限。
- **系统概览 · GPU 温度**：在第一张概览卡「处理器」下方新增显卡名称与实时温度（含负载），优先显示独显（NVIDIA / AMD），无需管理员。
- **系统概览 · 激活详情**：在激活状态下方新增版本（edition）、渠道（channel）、部分产品密钥、宽限期等详细信息（参考 `slmgr.vbs /dlv`）。
- **设置 · 开机自动启动**：可选登录 Windows 后自动启动（写入当前用户启动项，无需管理员）。
- **设置 · 托盘常驻图标**：可选无论窗口是否最小化都在系统托盘常驻图标；首次开启时弹出确认提示，并引导在 Windows 11 的「^」溢出区中找到图标。

### 调整
- **开发人员模式门控收紧**：整个「开发」栏，以及「系统」栏下的「系统维护 / 进程管理 / 启动项 / 环境变量」，现仅在开启开发人员模式后才显示（默认隐藏），避免普通用户误触系统级操作。
- **开启开发人员模式前新增风险二次确认弹窗**（适配应用主题色，三语本地化）。

### 删除
- 无。
